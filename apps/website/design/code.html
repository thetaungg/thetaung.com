Portfolio v1

React Gatsby GraphQL

The Problem: In a world of heavy client-side apps, I wanted a return to simplicity without losing modern DX. The goal was sub-100ms load times and a perfect Lighthouse score, proving that high interactivity doesn't have to come at the cost of performance.

The Solution: Built with Gatsby, the site leverages static site generation (SSG) to deliver pre-rendered HTML. I implemented a custom GraphQL layer to manage content from local Markdown files, creating a seamless CMS-free workflow that lives directly in the git repository.

          }
        }
      }
    }
  `)
  // Dynamic page generation logic...
}

Beyond the build architecture, I focused heavily on asset optimization. Images are processed at build time into modern formats (AVIF, WebP) with blur-up placeholders. The result is a site that feels instantaneous even on 3G networks.

Rick & Morty Wiki

This project is a deep dive into asynchronous JavaScript and state management. It consumes the public Rick and Morty API to build a rich, interactive wiki. The challenge was to handle complex data relationships without external libraries.

webhook
useCharacterSearch.js code 
const useCharacterSearch = (query) => {
  const [results, setResults] = useState([]);
  const [loading, setLoading] = useState(false);
  useEffect(() => {
     // Debounced API fetch logic to prevent
     // excessive network requests while typing
     fetchCharacters(query).then(data => {
        setResults(data);
        setLoading(false);
     });
  }, [query]);
  return { results, loading };
};

The application handles complex data fetching scenarios, including pagination, searching, and filtering by character status, species, and gender. I implemented a custom caching mechanism to minimize network requests for previously visited pages, effectively building a lightweight state management system from scratch.

The architecture adheres to the separation of concerns principle, organizing code into clear layers (routes, controllers, services, models). It comes pre-configured with essential security features like Helmet, CORS, and rate limiting. The starter also includes a robust error handling middleware that standardizes API responses across the application, making frontend integration predictable and smooth.
