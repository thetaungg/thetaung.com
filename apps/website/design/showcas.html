Project Overview

In a digital landscape dominated by heavy client-side applications, I sought a return to simplicity without sacrificing modern developer experience (DX). The objective for Portfolio v1 was clear: achieve sub-100ms load times and a perfect Lighthouse score, demonstrating that high interactivity doesn't have to compromise performance.

This project serves as both my professional calling card and a playground for experimenting with static site generation (SSG) techniques.

The Challenge

Most modern portfolios suffer from "bloat"—loading heavy JavaScript bundles for simple content display. My challenge was to architect a system that:

  • Loads instantly even on 3G networks.
  • Maintains a high level of visual fidelity with crisp typography and imagery.
  • Is easily maintainable without a complex CMS backend.
  • Is fully accessible (WCAG AA compliant).

Technical Approach

I chose Gatsby as the core framework because of its robust plugin ecosystem and image processing capabilities. By leveraging Gatsby's GraphQL data layer, I could decouple my content from the UI components completely.

1. Content Management via MDX

Instead of a database, content lives in Markdown (MDX) files within the Git repository. This allows for version-controlled content updates and the ability to embed React components directly inside blog posts.

gatsby-node.js 
exports.createPages = async ({ actions, graphql, reporter }) => {
  const { createPage } = actions
  // Query for markdown nodes to use in creating pages.
  const result = await graphql(`
    {
      allMdx {
        nodes {
          id
          fields { slug }
          internal { contentFilePath }
        }
      }
    }
  `)
  // Handle errors & create pages...
}

2. Image Optimization Strategy

Images are often the biggest bottleneck. I utilized gatsby-plugin-image to generate multiple sizes of every image at build time. The browser then serves the optimal size based on the device's viewport, with AVIF and WebP formats prioritized. A blurred placeholder is shown initially to prevent layout shift (CLS).
